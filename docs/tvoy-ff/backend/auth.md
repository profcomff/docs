# Аутентификация
Чтобы разрабатывать что-то в Твой ФФ, вам может понадобиться аутентификация.

## Первый шаг
Вам необходимо зарегестрироваться в Твой ФФ через почту. После этого вы, также, можете запросить доступы к платформе, если они вам нужны. Более подробно можно посмотреть в статье: <https://docs.profcomff.com/quick-start/dev/environment.html#forma-infrastrukturnyh-dostupov>

## Доступы к API
1. Зайдите по адресу: <https://app.test.profcomff.com/admin> или <https://app.profcomff.com/admin>
    Здесь будет отображаться ваш токен и его права, также тут видно ваш `id`. Более подробно про выбор окружения ниже.
2. Скопируйте этот токен, зайдите в Swagger UI, справа сверху есть кнопка `Authorize`, там в единственную строку ввода его надо вставить
3. Можете кидать любые запросы к API

## Как понять, откуда брать токен?
- Если вы разрабатываете локально любой проект, кроме `Auth API`, то токен надо брать отсюда: <https://app.test.profcomff.com/admin>
- Если вы разрабатываете локально `Auth API`, то токен надо брать из локальной ручки `/me`, которая находится в Swagger UI. До этого надо зарегистрироваться в своем локальном `Auth API`: это можно сделать либо по почте (ручка `/email/registration`), либо так:
    ```bash
    alembic upgrade head # skip if migration already passed
    make create-admin
    ```

    Получить токен можно так:
    
    ```bash
    make login-admin
    ```
    
    Логин и пароль тестового юзера написаны в `Makefile`
    Если у вас нет `make`, то зайдите в `Makefile` и копируйте команды, которые он вызывает. Либо поставьте `make`.
